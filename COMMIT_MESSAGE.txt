fix(security): move JWT token from localStorage to HttpOnly cookie (Bug #S3)

BREAKING CHANGE: JWT token authentication mechanism changed from localStorage to HttpOnly cookie

## Changes:

### Backend:
- Set JWT token via HttpOnly cookie in login/google endpoints
- Added /auth/logout endpoint to clear cookie
- Updated auth plugin to read token from cookie (fallback to Bearer)
- Cookie config: httpOnly=true, secure=production, sameSite=none (production)

### Frontend:
- Removed token from Zustand store (only user data persisted)
- Updated API client to send credentials (cookies) automatically
- Removed Authorization header (token now in cookie)
- Updated LoginPage to not store token in state
- Updated Sidebar logout to call API before clearing state

## Security Improvements:
- ✅ Token cannot be accessed by JavaScript (XSS protection)
- ✅ HttpOnly cookie prevents token theft via XSS
- ✅ Secure flag ensures HTTPS-only in production
- ✅ SameSite=none allows cross-origin (frontend ↔ backend)
- ✅ Backward compatible (supports Bearer token for testing)
